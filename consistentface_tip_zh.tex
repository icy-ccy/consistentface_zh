\begin{abstract}
严重退化会削弱人脸图像中的身份与属性线索，使现有方法虽能生成视觉自然的结果，却可能偏离目标身份。参考图像和文本提示可补充退化观测中缺失的信息，但参考图像可能过时或失配，文本提示也可能包含错误描述，二者还可能仅在局部属性上发生冲突。为此，本文提出 ConsistentFace，一种面向参考--文本冲突的可信度分配适配器。ConsistentFace 计算参考--文本、观测--参考、观测--文本和参考集合内部的一致性，并结合退化强度估计条件可信度。模型随后在全局和属性槽级分配参考、文本与回退条件。训练阶段，冲突类型仅用于监督条件分配；推理阶段无须提供冲突标签。大量实验表明，ConsistentFace 降低了条件冲突造成的身份差距和错误属性采纳率，并在自然语言、多评估器与人工评审中保持一致趋势；退化变化压力测试进一步揭示了文本冲突造成的身份下降。
\end{abstract}

\section{绪论}
\label{sec:main_intro}

盲人脸恢复旨在从受到未知退化影响的低质量人脸图像中恢复高质量人脸图像 \cite{wang2021gfpgan,wang2022restoreformer}。真实场景中的人脸图像常同时受到模糊、噪声、下采样和压缩等退化 \cite{wang2021gfpgan,wang2023dr2}。尽管现有方法能够改善视觉质量，严重退化下的身份线索仍可能不足，导致恢复结果偏离目标身份 \cite{suin2024diffuse,liu2025faceme,niu2026iconface}。

现有方法主要通过生成先验或外部条件补充退化输入中的信息。预训练人脸生成器、离散码本和扩散模型可为结构与细节恢复提供生成先验 \cite{wang2021gfpgan,zhou2022codeformer,lin2024diffbir}。另一些方法利用同身份参考图像补充身份相关细节 \cite{li2018gfrnet,li2020asffnet,liu2025faceme}，或通过文本信息提供语义线索与可控恢复 \cite{xie2023tfrgan,li2025mcs,zhu2026a2bfr}。本文进一步关注参考与文本条件不一致时的条件选择问题。

然而，外部条件未必适用于当前观测。退化图像与目标人脸直接对应，但其中可辨认的身份与属性线索会随退化增强而减弱 \cite{liu2025faceme,niu2026iconface}。参考图像可能因姿态、表情、光照、年龄或局部面部状态差异而提供不适用的外观信息 \cite{li2018gfrnet,li2020asffnet,niu2026iconface}；自动描述模型或视觉语言模型也可能生成与图像不一致的描述 \cite{rohrbach2018object,li2023pope}。为此，本文将过时参考、身份失配、文本错误和局部属性冲突纳入统一设置，并以退化观测为依据估计参考与文本条件的可信度及其适用粒度；当两类条件的证据均不足时，模型采用回退条件。

本文提出 ConsistentFace，一种面向参考--文本冲突的可信度分配适配器。给定退化观测 $y$、参考集合 $\mathcal{R}$ 与文本提示 $T$，该适配器在参考和文本 token 进入去噪网络前估计其相对可信度，并在全局与属性槽级分配两类条件；当证据不足时，模型转而使用回退条件。该设计无须改动扩散主干。

图~\ref{fig:conflict_motivation} 展示了四类典型冲突：文本幻觉、错误或过时参考、局部属性冲突以及两类外部条件同时失效。与固定拼接或平均不同，ConsistentFace 根据冲突来源和作用范围调整参考、文本与回退条件的贡献。

该任务面临三个挑战：1）\textbf{冲突来源判别。} 相同的低一致性可能由文本错误、过时参考或身份失配引起；2）\textbf{局部与全局条件解耦。} 局部属性冲突不应导致其余可靠条件被整体抑制；3）\textbf{训练与推理一致性。} 冲突类型标签仅在训练阶段可用，模型在推理时需要从输入本身判断条件可信度。

针对上述挑战，ConsistentFace 采用三项对应设计。首先，模型联合建模参考--文本一致性、观测--参考一致性、观测--文本一致性、参考集合内部一致性和退化强度，以判断冲突来源。其次，模型将文本划分为五类属性槽，并在各槽内独立分配参考、文本与回退条件，以隔离局部冲突。最后，冲突类型标签仅用于监督条件分配；推理时，模型依据 $(y,\mathcal{R},T)$ 完成分配，无须用户指定错误类型。

\textbf{本文的主要贡献如下：}

\begin{itemize}
    \item \textbf{提出冲突感知双先验恢复设定：} 结合参考--文本一致性、观测--条件一致性、参考集合内部一致性和退化强度，估计参考图像与文本提示的相对可信度。
    
    \item \textbf{设计全局与属性槽级条件分配机制：} 分别处理整体条件失效和局部属性冲突，并在证据不足时使用回退条件。

    \item \textbf{构建条件分配的训练与评测方案：} 通过排序与回退监督学习条件分配，并利用去相关约束降低提示模板和参考来源对分配结果的影响；推理阶段无须冲突标签。同一主干比较、反事实实验、多评估器验证和人工评审验证了方法的有效性。

\end{itemize}


\section{相关工作}
\label{sec:main_related}
\subsection{盲人脸恢复与生成先验}
现有盲人脸恢复方法主要利用生成先验补偿退化造成的结构与细节损失 \cite{wang2021gfpgan,zhou2022codeformer,lin2024diffbir}。GFPGAN 引入预训练人脸生成器先验 \cite{wang2021gfpgan}；RestoreFormer、VQFR 和 CodeFormer 分别利用高质量键值字典、向量量化字典和离散码本完成恢复 \cite{wang2022restoreformer,gu2022vqfr,zhou2022codeformer}。

近期方法进一步引入扩散先验。DR2、DifFace 和 DiffBIR 分别通过鲁棒退化去除、后验迭代恢复和潜空间生成先验改善恢复结果 \cite{wang2023dr2,yue2022difface,lin2024diffbir}。相关研究还将扩散先验用于真实图像超分辨率，并探索区域自适应、幻觉校正、视觉风格、3D 先验和参考梯度引导 \cite{wang2024stablesr,suin2024diffuse,gao2024diffmac,zhu2024visualstyle,lu2024threepriors,min2024pdgrad}。

尽管生成先验改善了恢复结果的感知质量和面部细节，严重退化仍可能使身份关键线索缺失，导致恢复结果偏离目标身份 \cite{suin2024diffuse,liu2025faceme,niu2026iconface}。为增强身份和属性一致性，现有研究进一步引入外部身份或语义条件 \cite{li2018gfrnet,liu2025faceme,xie2023tfrgan,an2026tbfr}。

\subsection{参考驱动与个性化人脸恢复}
为了增强身份一致性，参考驱动方法向恢复网络提供同一身份的额外图像。GFRNet 对引导图像进行变形，使其姿态和表情与退化观测对齐；ASFFNet 进一步进行空间对齐、光照变换和多参考特征的自适应融合 \cite{li2018gfrnet,li2020asffnet}。DMDNet 通过通用字典和身份特定字典分别存储一般人脸先验与身份相关特征 \cite{li2022dmdnet}。由于退化观测与参考图像可能存在姿态、光照和表情差异，参考信息的使用依赖适当的对齐或选择机制 \cite{li2018gfrnet,li2020asffnet,li2022dmdnet}。

另一类研究借鉴个性化生成方法。MyStyle 学习面向特定身份的个性化生成先验 \cite{nitzan2022mystyle}；Textual Inversion、DreamBooth 和 Custom Diffusion 通过学习文本嵌入或调整模型参数表示特定主体 \cite{gal2023textualinversion,ruiz2023dreambooth,kumari2023customdiffusion}。IP-Adapter 提供通用图像提示接口，PhotoMaker、InstantID 和 PuLID 则面向人像身份定制 \cite{ye2023ipadapter,li2024photomaker,wang2024instantid,guo2024pulid}。

FaceMe 与本文的任务设定较为接近。该方法使用身份编码器提取参考图像中的身份特征，并通过两阶段训练平衡低质量输入与参考图像之间的依赖，更换身份时无须重新微调 \cite{liu2025faceme}。FaceMe 假设参考图像与目标具有相同身份；ConsistentFace 进一步研究参考失效以及参考图像与文本提示发生冲突的情形。

\subsection{语言引导恢复与可变属性控制}
自然语言可作为面向用户的条件接口，为恢复过程提供内容语义或恢复要求 \cite{conde2024instructir,qi2024spire}。TFRGAN 将文本与图像信息融合到 StyleGAN 潜空间，并利用文本特征调制解码过程 \cite{xie2023tfrgan}；TBFR 构建了自然语言引导的盲人脸恢复方法与数据集 \cite{an2026tbfr}。MCS 支持由不同文本提示控制的多样化恢复结果 \cite{li2025mcs}；A$^2$BFR 通过图文交叉模态注意力和属性感知学习实现细粒度提示控制 \cite{zhu2026a2bfr}。

图像描述模型和大型视觉语言模型可能生成输入图像中不存在的对象 \cite{rohrbach2018object,li2023pope}。本文还将错误历史标注和不准确用户描述视为文本失效的可能来源，并在文本条件参与恢复前评估其与目标观测的一致性，同时区分整体语义偏差与局部属性冲突。

\subsection{多模态冲突与可信度分配}
当参考图像与文本提示同时作为恢复条件时，模型需要协调两类信息。ControlNet 通过附加网络引入空间控制，IP-Adapter 通过解耦交叉注意力分别处理图像与文本提示，PAIR Diffusion 利用多模态无分类器引导组合参考图像与文本条件 \cite{zhang2023controlnet,ye2023ipadapter,goel2024pairdiffusion}；CLIP 表征可用于衡量图文语义一致性 \cite{radford2021clip}。

近期研究进一步协调身份、结构与文本条件。CoFaDiff 研究个性化生成中身份一致性与文本可编辑性的协调 \cite{ming2026cofadiff}；IConFace 以参考图像提供全局身份锚点，并利用退化图像提供空间结构锚定，从而统一有参考与无参考人脸恢复 \cite{niu2026iconface}。本文则关注参考身份条件与文本条件发生冲突时的相对可信度分配。

\section{问题表述}

给定清晰目标人脸 $x$，其退化观测记为 $y=\mathcal{D}(x)$，其中未知退化算子 $\mathcal{D}$ 可包含模糊、下采样、噪声和压缩 \cite{wang2021gfpgan,wang2023dr2}。模型同时接收参考图像集合 $\mathcal{R}$ 和自然语言提示 $T$，并输出恢复结果 $\hat{x}=G_{\theta}(y,\mathcal{R},T)$。本文将 $y$ 作为观测锚点，即它与目标人脸直接对应，但其中的身份和属性线索可能因退化而不足；参考图像和文本提示则作为需要评估的外部条件。

本文将自然语言提示解析为属性槽集合 $\{T_a:a\in\mathcal{A}\}$ 和自由文本，并选取五类可解析属性构造局部冲突。该属性集合用于验证槽级条件分配，不覆盖全部面部语义。

模型需要完成三类条件分配：在参考条件 token 和文本语义 token 之间确定全局偏好；当两类外部条件均可疑时降低其权重；在局部属性槽内独立处理冲突。本文以 $\alpha$ 表示全局参考--文本偏好，以 $c$ 和 $1-c$ 分别表示外部条件使用权重和回退权重；$\alpha_a$、$c_a$ 和 $1-c_a$ 表示属性槽 $a$ 中对应的局部权重。

训练阶段，每个样本带有冲突类型标签 $r$，用于构造全局与槽级排序对、回退目标和受影响的属性槽集合。推理阶段不使用该标签，模型直接根据 $(y,\mathcal{R},T)$ 预测条件权重。

\section{方法}
\label{sec:main_method}

\subsection{方法概述}
ConsistentFace 以 SDXL 和 ControlNet 为恢复主干 \cite{podell2023sdxl,zhang2023controlnet}，并在参考与文本条件进入去噪网络前加入可信度分配适配器。退化图像 $y$ 由 ControlNet 编码为结构条件，参考集合 $\mathcal{R}$ 和文本提示 $T$ 分别编码为参考条件 token 和语义 token。

模型首先提取三类输入的条件表征，再计算多源证据，最后生成全局、属性槽级和回退条件 token。恢复主干根据分配后的条件完成去噪。

完整数据流如图~\ref{fig:overall_architecture} 所示：条件编码、多源证据和可信度分配位于扩散恢复主干之前，分配后的条件 token 与 ControlNet 结构条件共同指导去噪。

\subsection{条件编码}
对于参考集合 $\mathcal{R}=\{R_i\}_{i=1}^{M}$，模型分别提取身份特征和视觉特征，并通过可学习映射器将多张参考图像聚合为参考条件 token：
\begin{equation}
\label{eq:reference_tokens}
\begin{aligned}
e^{I}_{R_i}&=E_I(R_i),\\
Z_{R_i}&=E_V(R_i),\\
V&=\Phi_R\!\left(\{e^{I}_{R_i},Z_{R_i}\}_{i=1}^{M}\right),\\
V&=[v_1,\ldots,v_K].
\end{aligned}
\end{equation}
其中，$E_I$ 是身份编码器（实现中采用 ArcFace \cite{deng2019arcface}），$E_V$ 复用 PhotoMaker 的 CLIP 视觉编码器 \cite{li2024photomaker}，$\Phi_R$ 是参考特征映射器。$V=[v_1,\ldots,v_K]$ 表示融合身份与视觉特征的参考条件 token，$K$ 为 token 数量。该 token 序列将在后续全局和属性槽级分配中分别加权。

文本提示 $T$ 使用 SDXL 文本编码器 \cite{podell2023sdxl} 得到序列语义特征，并映射到与参考条件 token 兼容的条件空间：
\begin{equation}
\label{eq:text_tokens}
\begin{aligned}
H&=E_T(T),\\
t&=\operatorname{Pool}(H),\\
U&=\Phi_T(H)=[u_1,\ldots,u_K].
\end{aligned}
\end{equation}
其中，$H$ 是文本序列特征，$t$ 是全局文本语义向量，$U=[u_1,\ldots,u_K]$ 是文本语义 token。$\Phi_T$ 将文本 token 映射到与 $V$ 相同的维度和 token 数量。

退化图像由编码器 $E_y$ 映射为观测特征，参考条件 token 的平均表示定义为
\begin{equation}
\label{eq:mean_tokens}
f_y=E_y(y),\qquad
\bar{v}=\frac{1}{K}\sum_{k=1}^{K}v_k.
\end{equation}
其中，$f_y$ 是退化观测特征，$\bar{v}$ 是参考条件 token 的平均表示。$f_y$ 用于计算退化观测与参考、文本条件的一致性。

\subsection{多源一致性与退化证据}
单独使用参考--文本相似度难以判断不一致来自哪一方。为此，ConsistentFace 联合计算参考--文本一致性、观测--参考一致性、观测--文本一致性、参考集合内部一致性和退化强度。余弦相似度定义为
\begin{equation}
\label{eq:cosine_similarity}
\operatorname{sim}(a,b)=
\frac{a^{\top}b}{\|a\|_2\,\|b\|_2}.
\end{equation}
该相似度用于比较投影到同一空间中的条件特征，取值越大表示二者越一致。

ConsistentFace 构造四类全局一致性分数：
\begin{subequations}
\label{eq:multi_evidence_scores}
\begin{align}
s_{RT}
&=\operatorname{sim}\!\left(P_R(\bar{v}),P_T(t)\right),\\
s_{yR}
&=\operatorname{sim}\!\left(P_y^{I}(f_y),P_R(\bar{v})\right),\\
s_{yT}
&=\operatorname{sim}\!\left(P_y^{A}(f_y),P_T^{A}(t)\right),\\
s_{RR}
&=\frac{2}{M(M-1)}
\sum_{1\leq i<j\leq M}
\operatorname{sim}\!\left(e^{I}_{R_i},e^{I}_{R_j}\right).
\end{align}
\end{subequations}
其中，$s_{RT}$ 衡量参考与文本的一致性，$s_{yR}$ 衡量参考身份与退化观测中身份线索的一致性，$s_{yT}$ 衡量文本属性与退化观测中可见属性的一致性，$s_{RR}$ 衡量参考图像间的身份一致性。$P_R$、$P_T$、$P_y^{I}$、$P_y^{A}$ 和 $P_T^{A}$ 是可学习投影头，用于将不同来源的特征映射到可比较空间。若只有一张参考图像，$s_{RR}$ 设为中性常数，避免该项主导分配结果。

除一致性证据外，模型还估计退化强度：
\begin{equation}
\label{eq:degradation_score}
q=Q(y).
\end{equation}
其中，$Q$ 为退化强度估计器。$q$ 越大，表示退化观测中可用于核验身份和属性的线索越弱。$q$ 与一致性分数共同输入条件分配器。

最终，适配器将原始分数和差值分数组成全局证据向量：
\begin{equation}
\label{eq:global_evidence_vector}
g=
\left[
s_{RT},\,
s_{yR},\,
s_{yT},\,
s_{RR},\,
q,\,
s_{RT}-s_{yR},\,
s_{RT}-s_{yT}
\right].
\end{equation}
两个差值分别补充身份侧和属性侧的方向信息，以辅助判断不一致主要来自参考还是文本。

\subsection{全局条件分配}
全局分配器确定参考条件 token、文本语义 token 和回退 token 的相对贡献。给定证据向量 $g$，分配器预测两个标量权重：
\begin{subequations}
\label{eq:global_weights}
\begin{align}
\alpha&=\sigma\!\left(f_{\alpha}(g)\right),\\
c&=\sigma\!\left(f_{c}(g)\right).
\end{align}
\end{subequations}
其中，$f_{\alpha}$ 和 $f_c$ 是两个多层感知机，$\sigma(\cdot)$ 将输出压缩到 $[0,1]$。$\alpha$ 控制参考条件 token 与文本语义 token 的相对分配，$c$ 表示外部条件使用权重，$1-c$ 表示全局回退权重。参考更可靠时，$\alpha$ 增大；文本更可靠时，$\alpha$ 减小；两类外部条件均可疑时，$c$ 减小。

全局条件 token 因此写为
\begin{equation}
\label{eq:global_fusion}
F_g=
c\Big(\alpha V+(1-\alpha)U\Big)
+(1-c)B_g.
\end{equation}
其中，$B_g$ 是与 $V$ 和 $U$ 维度相同的可学习回退 token。随着 $c$ 减小，$B_g$ 的相对贡献增大。

\subsection{属性槽级条件分配}
全局分配难以处理局部属性冲突。例如，眼镜描述可能出错，而发型和遮挡描述仍然可用。本文将提示拆分为属性槽集合 $\mathcal{A}$，包括头发、眼镜/眼睛、胡须/嘴巴、年龄/皮肤和配饰/遮挡五类属性。对于属性槽 $a\in\mathcal{A}$，模型从文本序列中提取槽级特征：
\begin{equation}
\label{eq:slot_text_feature}
t_a=\Psi_a(H).
\end{equation}
其中，$\Psi_a$ 是属性槽 $a$ 的文本特征提取器，$t_a$ 表示该槽对应的文本语义 token。

模型使用可学习查询对参考条件 token 执行缩放点积注意力汇聚，以提取对应属性槽的参考特征 \cite{vaswani2017attention}：
\begin{subequations}
\label{eq:slot_attention}
\begin{align}
\omega_{a,k}
&=
\frac{
\exp\!\left((W_Q e_a)^{\top}(W_K v_k)/\sqrt{d}\right)
}{
\sum_{\ell=1}^{K}
\exp\!\left((W_Q e_a)^{\top}(W_K v_{\ell})/\sqrt{d}\right)
},\\
v_a
&=
\sum_{k=1}^{K}\omega_{a,k}W_V v_k.
\end{align}
\end{subequations}
其中，$e_a$ 是属性槽查询向量，$W_Q$、$W_K$ 和 $W_V$ 分别为查询、键和值的线性投影矩阵，$v_a$ 表示与属性槽 $a$ 对应的参考视觉特征。

适配器随后计算槽内参考--文本一致性和观测--文本一致性：
\begin{subequations}
\label{eq:slot_scores}
\begin{align}
s_a
&=\operatorname{sim}\!\left(P_R^{a}(v_a),P_T^{a}(t_a)\right),\\
s_{ya}
&=\operatorname{sim}\!\left(P_y^{a}(f_y),P_T^{a}(t_a)\right).
\end{align}
\end{subequations}
其中，$s_a$ 表示参考特征与文本特征的一致性，$s_{ya}$ 表示文本槽与退化观测中可见属性的一致性。

每个属性槽使用局部证据预测参考--文本分配权重和外部条件使用权重：
\begin{subequations}
\label{eq:slot_weights}
\begin{align}
g_a&=\left[g,\,s_a,\,s_{ya},\,\alpha,\,c\right],\\
\alpha_a&=\sigma\!\left(f_{\alpha}^{a}(g_a)\right),\\
c_a&=\sigma\!\left(f_{c}^{a}(g_a)\right).
\end{align}
\end{subequations}
其中，$g_a$ 拼接了全局证据、槽级一致性分数和全局分配结果。$\alpha_a$ 控制该槽中参考视觉特征与文本特征的相对分配，$c_a$ 表示该槽的外部条件使用权重，$1-c_a$ 表示相应的回退权重。

槽级融合 token 定义为
\begin{equation}
\label{eq:slot_fusion}
F_a=
c_a\Big(\alpha_a v_a+(1-\alpha_a)t_a\Big)
+(1-c_a)B_a.
\end{equation}
其中，$B_a$ 是属性槽 $a$ 的回退 token。所有条件 token 最终拼接为
\begin{equation}
\label{eq:condition_tokens}
F=\operatorname{Concat}\!\left(F_g,\{F_a:a\in\mathcal{A}\}\right).
\end{equation}
其中，$F$ 是注入扩散模型的条件 token 集合。该设计使局部错误主要影响对应属性槽，从而减少其对其他文本条件的干扰。

图~\ref{fig:reliability_allocation} 进一步给出全局与属性槽级分配器的内部结构及三类典型路由行为。全局分配器控制外部条件的整体偏好和置信度，槽级分配器则把局部冲突限制在相应属性槽内。

\caption{全局与属性槽级可信度分配。全局分配器预测参考--文本偏好 $\alpha$ 和外部条件置信度 $c$；属性槽分配器进一步预测 $\alpha_a$ 和 $c_a$，使局部冲突主要影响对应属性槽，而不整体抑制其余可靠信息。}

\subsection{训练目标与推理策略}
训练目标由扩散去噪损失、可信度分配监督和去相关约束组成。可信度分配监督包括全局与槽级排序损失以及回退损失。遵循扩散模型和潜空间扩散的噪声预测训练形式 \cite{ho2020ddpm,rombach2022ldm,podell2023sdxl}，设 $z_0$ 为清晰目标图像 $x$ 的 VAE 潜变量，$\tau$ 为扩散时间步，$\epsilon$ 为标准高斯噪声，则加噪潜变量为
\begin{equation}
\label{eq:noisy_latent}
z_{\tau}
=
\sqrt{\bar{\gamma}_{\tau}}\,z_0
+
\sqrt{1-\bar{\gamma}_{\tau}}\,\epsilon.
\end{equation}
其中，$\bar{\gamma}_{\tau}$ 表示扩散噪声调度在时间步 $\tau$ 的累计保留系数。

对应的去噪损失为
\begin{equation}
\label{eq:diffusion_loss}
\mathcal{L}_{\mathrm{diff}}
=
\mathbb{E}_{z_0,\epsilon,\tau}
\left[
\left\|
\epsilon-\epsilon_{\theta}(z_{\tau},\tau,C_y,F)
\right\|_2^2
\right].
\end{equation}
其中，$C_y$ 是由退化图像 $y$ 提取的 ControlNet 结构条件，$F$ 是可信度分配后的条件 token。

冲突标签 $r$ 用于构造权重排序对和回退目标，推理阶段不使用该标签。令全局权重集合为
\begin{equation}
\label{eq:global_weight_set}
\mathcal{W}_g=\{\alpha,\ 1-\alpha,\ c,\ 1-c\},
\end{equation}
其中四项分别对应参考偏好、文本偏好、外部条件使用权重和回退权重。每个冲突类型 $r$ 被映射到一组期望排序对 $\mathcal{P}_g(r)\subseteq\{(\alpha,1-\alpha),(1-\alpha,\alpha),(c,1-c),(1-c,c)\}$。文本幻觉要求参考权重大于文本权重，错误或过时参考要求文本权重大于参考权重；当参考与文本均不可靠时，回退权重大于外部条件使用权重。全局排序损失为
\begin{equation}
\label{eq:global_rank_loss}
\begin{aligned}
\mathcal{L}^{g}_{\mathrm{rank}}
=&
\frac{1}{|\mathcal{P}_g(r)|}
\sum_{(w^+,w^-)\in\mathcal{P}_g(r)}
\\[-1pt]
&\max\!\left(0,\ m_g-(w^+-w^-)\right).
\end{aligned}
\end{equation}
其中 $m_g$ 是全局排序间隔。该损失约束权重的相对方向和间隔，无须为各类冲突预设固定权重；当 $\mathcal{P}_g(r)$ 为空时，该项置为 $0$。

属性槽权重同样采用成对间隔排序损失。对于受冲突影响的属性槽 $a\in\mathcal{A}^{-}(r)$，标签 $r$ 指定参考、文本或回退权重的期望顺序。令槽级权重集合为
\begin{equation}
\label{eq:slot_weight_set}
\mathcal{W}_a=\{\alpha_a,\ 1-\alpha_a,\ c_a,\ 1-c_a\}.
\end{equation}
槽级排序对 $\mathcal{P}_a(r,a)$ 仅在 $a\in\mathcal{A}^{-}(r)$ 时启用；无冲突槽不施加显式排序约束，以保留仍可信的文本指导。槽级排序损失为
\begin{equation}
\label{eq:slot_rank_loss}
\begin{aligned}
\mathcal{L}^{a}_{\mathrm{rank}}
=&
\frac{1}{|\mathcal{A}^{-}|}
\sum_{a\in\mathcal{A}^{-}}
\frac{1}{|\mathcal{P}_a(r,a)|}
\\[-1pt]
&\sum_{(w_a^+,w_a^-)\in\mathcal{P}_a(r,a)}
\max\!\left(0,\ m_a-(w_a^+-w_a^-)\right).
\end{aligned}
\end{equation}
其中 $m_a$ 是槽级排序间隔。当 $\mathcal{A}^{-}$ 为空时，该项置为 $0$。

回退损失监督模型在全局和属性槽级是否启用回退。设 $b_g(r)\in\{0,1\}$ 表示全局是否需要回退，$b_a(r,a)\in\{0,1\}$ 表示属性槽是否需要回退，则
\begin{equation}
\label{eq:fallback_loss}
\begin{aligned}
\mathcal{L}_{\mathrm{fb}}
=&
-b_g\log(1-c)-(1-b_g)\log c\\
&+
\frac{1}{|\mathcal{A}|}
\sum_{a\in\mathcal{A}}
\left[-b_a\log(1-c_a)-(1-b_a)\log c_a\right].
\end{aligned}
\end{equation}
实现中对 $\log(\cdot)$ 的输入加入 $10^{-6}$ 下界以避免数值不稳定。

本文进一步采用去相关约束，以降低证据向量与提示模板、参考来源标识之间的批内线性相关性。令 $\tilde{g}$ 和 $\tilde{g}_a$ 表示批内标准化后的全局与槽级证据，$\xi$ 为提示模板与参考来源标识的一热编码拼接，则
\begin{equation}
\label{eq:decorrelation_loss}
\mathcal{L}_{\mathrm{decorr}}
=
\left\|\operatorname{Cov}(\tilde{g},\xi)\right\|_F^2
+
\frac{1}{|\mathcal{A}|}
\sum_{a\in\mathcal{A}}
\left\|\operatorname{Cov}(\tilde{g}_a,\xi)\right\|_F^2.
\end{equation}
该约束用于减少条件分配对特定提示模板和参考来源的依赖。

完整训练目标为
\begin{equation}
\label{eq:training_objective}
\mathcal{L}
=
\mathcal{L}_{\mathrm{diff}}
+
\lambda_g\mathcal{L}^{g}_{\mathrm{rank}}
+
\lambda_a\mathcal{L}^{a}_{\mathrm{rank}}
+
\lambda_b\mathcal{L}_{\mathrm{fb}}
+
\lambda_d\mathcal{L}_{\mathrm{decorr}}.
\end{equation}
其中，$\lambda_g$、$\lambda_a$、$\lambda_b$ 和 $\lambda_d$ 分别控制全局排序损失、槽级排序损失、回退损失和去相关约束。除移除相应损失项的消融变体外，其余 ConsistentFace 变体采用相同的损失权重。超参数在验证集上确定，并在后续测试中固定。

推理时不使用冲突标签。模型根据 $(y,\mathcal{R},T)$ 计算 $g$ 和 $g_a$，并生成最终条件 $F$。

训练监督和无标签推理的关系如图~\ref{fig:training_inference} 所示。冲突类型仅用于训练阶段构造排序对、回退目标和去相关约束；部署时模型只接收 $y$、$\mathcal{R}$ 与 $T$。

\section{实验}
\label{sec:main_experiments}

\subsection{实验设置}
\subsubsection{训练数据与实现细节}
训练集包含 70,000 条 FFHQ 目标记录 \cite{liu2025faceme}，每条记录均配有语义提示和由 FFHQRef 提取的同身份参考特征。所有图像缩放至 $512\times512$，每次随机采样不超过四张参考图像。提示模板为 ``A photo of face. \{semantic\}''；后续训练阶段以 $0.1$ 的概率移除身份 token 替换，以保留仅使用文本条件的基础能力。

模型采用三个训练阶段：第一阶段联合训练参考映射、ControlNet 和可信度分配适配器 100 个 epoch，文本错误、参考错误和双条件失效的采样概率均为 $0.20$；第二阶段固定参考映射与适配器，继续训练 ControlNet 30 个 epoch；第三阶段以 $10^{-5}$ 的学习率联合微调 ControlNet 和适配器 10 个 epoch，三类冲突的采样概率分别为 $0.20$、$0.20$ 和 $0.15$。训练使用 batch size 4 和 bf16 混合精度。除特别说明外，推理采用固定噪声时间步 799、40 步去噪、CFG 4.0、ControlNet 条件强度 1.1 和小波颜色校正。

同一主干实验固定 SDXL/ControlNet 恢复主干 \cite{podell2023sdxl,zhang2023controlnet}、参考编码器、训练清单和推理参数。其中的所有可训练双先验变体均接收 $(y,\mathcal{R},T)$，并使用五个随机种子训练和评测。

\subsubsection{测试数据与冲突构造}
本文沿用 FaceMe 的 LFW-Test、WebPhoto-Test 和 WIDER-Test 划分 \cite{liu2025faceme}，并在各划分上构造匹配与冲突记录。三者分别包含 1,711/1,707、407/407 和 970/969 条匹配/冲突记录，并按照可用身份标识与训练集去重。真实退化测试包括 640 张低照度或社交媒体图像（320 个身份）和 520 张老照片或强压缩图像（260 个身份）。每个身份均配有同身份参考图像；提示由注释者在不查看恢复结果的条件下撰写，标注分歧通过第二轮复核解决。

退化过程包含模糊、下采样、噪声和 JPEG 压缩 \cite{wang2021gfpgan,zhou2022codeformer,wang2023dr2}。在同一目标身份、退化观测和参考集合上，本文构造匹配提示、轻度与重度文本冲突、文本幻觉、过时参考、错误参考以及双条件失效。文本幻觉提示来自其他身份，且不参与阈值或检查点选择。自由文本子集进一步覆盖跨年龄参考、遮挡、少见配饰和多个局部属性同时冲突等情形。

为区分文本错误与参考错误，本文构造 $s_{RT}$ 和退化强度 $q$ 分布匹配的配对反事实测试集，并平衡退化类型、参考数量、参考质量、身份划分和提示长度。冲突标签仅用于训练监督和评测分组，推理阶段不提供该标签。

\subsubsection{比较方法与评价指标}
外部比较包括通用盲人脸恢复、参考驱动恢复和文本驱动恢复方法；同一主干比较则固定输入、参数规模和训练预算，只改变条件融合或可信度分配模块。该设置用于降低额外输入和恢复主干差异对路由机制比较的混淆。

恢复质量采用 PSNR、SSIM \cite{wang2004ssim} 和 LPIPS \cite{zhang2018lpips}。身份指标分别由 ArcFace、AdaFace、MagFace 和 CurricularFace 评测 \cite{deng2019arcface,kim2022adaface,meng2021magface,huang2020curricularface}，主表报告四个识别器的平均身份差距。属性指标包括 CelebA 属性分类器 \cite{liu2015faceattributes}、OpenCLIP 属性评分 \cite{radford2021clip,ilharco2021openclip} 和开放属性解析器。TAR 降幅衡量冲突条件造成的验证性能下降，排名反转表示传统恢复指标选择的候选在人脸身份或目标属性上反而更差。指标公式和判定细节见附录~\ref{app:metric_definitions}；所有阈值均由验证集确定，并在测试阶段固定。

\section{局限性与伦理说明}
\label{sec:main_limitations}

ConsistentFace 用于分配参考图像和文本提示，不提供身份认证能力。当退化严重、关键区域被遮挡或输入仅保留微弱的身份与属性线索时，模型可能无法获得足够证据来校准两类条件。当前实现还受到属性槽和训练清单的限制，五类属性槽不能完整描述表情、妆容、细粒度肤色、脸型、痣疤、复杂光照或多人交互等开放属性。完整模型仍有 44.6\% 的排名反转率，说明传统画质指标与身份和属性评价之间仍存在不一致。后续工作可进一步研究细粒度属性标注、遮挡感知身份置信度、时间感知参考加权和不确定性校准。

生成式人脸恢复依赖生成先验补充退化输入中难以辨认的细节 \cite{wang2021gfpgan,zhou2022codeformer,lin2024diffbir}，其输出不应被视为原始细节的事实性还原或身份认证证据。ConsistentFace 仅用于经授权的照片修复、媒体增强和恢复算法评测，不建议用于门禁、执法、金融开户、招聘筛查或身份核验等高风险场景。面部识别系统还伴随隐私和准确性风险 \cite{nist2019frt,gao2020facialrecognition}。

模型或数据发布应附带输入退化记录、参考来源摘要、可信度诊断和失败模式说明。真实退化切片在本文中仅用于压力诊断；发布相应结果时还应同步提供来源与授权范围、身份级划分清单和去重哈希，以便复核。对于受许可、肖像权或隐私约束的人脸数据，宜发布提示、评测清单、哈希值、聚合统计和可复现实验脚本。涉及未成年人、敏感身份或不可撤回同意的数据不应随模型发布原始图像。

\section{结论}
\label{sec:main_conclusion}

本文提出 ConsistentFace，用于处理身份保持人脸恢复中的参考--文本条件冲突。该方法结合参考--文本、观测--参考、观测--文本和参考集合内部的一致性以及退化强度，在全局和属性槽级分配参考、文本与回退条件，推理时无须冲突标签。同一主干、消融和反事实实验表明，ConsistentFace 降低了冲突条件下的身份差距与错误属性采纳率；自然语言、多评估器和人工评审进一步验证了这一趋势，退化变化压力测试则揭示了文本冲突在分布变化下造成的身份下降。

\section{指标定义与统计协议}
\label{app:metric_definitions}

给定身份评估器 $h$ 的相似度 $S_h$，身份差距定义为
\begin{equation}
\begin{aligned}
\mathrm{IdentityGap}=\mathbb{E}_{h}\big[
&\mathbb{E}_{\mathrm{match}}S_h(\hat{x}^{\mathrm{match}},x)\\
&-\mathbb{E}_{\mathrm{conf}}S_h(\hat{x}^{\mathrm{conf}},x)
\big].
\end{aligned}
\end{equation}
其中，$\mathrm{match}$ 和 $\mathrm{conf}$ 分别表示匹配条件和对应冲突条件。对于候选集 $\mathcal{C}_i$，排名反转定义为
\begin{equation}
\begin{aligned}
\mathrm{Rev}_i
&=\mathbf{1}\{H(k^\star)<H(j)-\tau,\ \exists j\in\mathcal{C}_i\},\\
k^\star
&=\arg\max_{k\in\mathcal{C}_i}M(k),
\end{aligned}
\end{equation}
其中，$M\in\{\mathrm{PSNR},\mathrm{SSIM},-\mathrm{LPIPS}\}$，$H$ 表示身份或目标属性质量，$\tau=10^{-4}$ 为验证集上固定的平局边距。属性槽 $a$ 上的错误属性采纳量为
\begin{equation}
\mathrm{WPA}_a=\mathbf{1}\{E_a(\hat{x})=z_a^{\mathrm{wrong}}\land E_a(\hat{x})\ne z_a^{\mathrm{target}}\}.
\end{equation}
所有阈值在验证集上确定，并在测试阶段保持不变。五种子实验报告均值；置信区间通过身份级配对 bootstrap 计算，多重比较采用 Holm 校正。
