\begin{abstract}
盲人脸恢复在严重退化下缺少足够上下文，容易生成视觉合理但身份偏移的结果；参考图像可补充身份线索，文本提示可描述发型、眼镜、胡须、年龄/皮肤和遮挡等可变属性，但二者在真实使用中可能过时、误匹配或携带自动描述幻觉。本文提出 ConsistentFace，一种面向参考--文本双先验冲突的人脸恢复可信度分配适配器。给定退化观测、参考集合和文本提示，ConsistentFace 提取参考--文本一致性、退化--参考身份一致性、退化--文本属性一致性、参考集合内部一致性和退化强度等证据，并预测全局参考/文本偏好、属性槽级偏好和保守回退置信度，使模型在参考可靠、文本可靠、二者均可疑或仅局部属性冲突时采用不同条件路径。训练阶段，本文使用冲突类型构造相对排序监督和回退正则；推理阶段不使用冲突标签。跨范式基准、同一主干比较、强双先验代理、属性槽消融、匹配-$s_{RT}$ 反事实划分、跨评估器验证和人工评审表明，ConsistentFace 的收益主要来自结构化可信度判断，而不是额外输入源、身份适配器强度或模型容量。
\end{abstract}

\section{绪论}
\label{sec:main_intro}

人脸恢复旨在从低质量观测中恢复清晰、自然且身份一致的面部图像。真实照片常同时受到模糊、噪声、压缩、低分辨率和光照变化影响，使一个退化输入对应多个可能的高清结果。近年来，DiffBIR、StableSR 等基于扩散先验的方法 \cite{lin2024diffbir,wang2024stablesr} 显著提升了视觉质量，但强生成先验并不能保证身份正确。当眼部结构、发际线、皱纹或年龄痕迹已在观测中丢失时，模型可能生成逼真的细节，却偏离真实身份。

一种自然的做法是引入外部上下文。参考图像可提供同一身份的外观线索，文本提示可描述发型、配饰、遮挡、年龄和皮肤状态等局部属性。已有参考引导、文本引导和个性化扩散方法大多围绕一个目标展开：如何更充分地注入这些先验。然而，双先验场景中的困难不只是“先验不够强”，还包括“先验是否可信”。参考图像可能来自不同年龄、不同域或错误身份；文本提示可能由自动描述模型生成，携带事实错误；二者还可能只在某个局部属性上冲突。

因此，本文关注的问题不是继续增加条件源，而是判断条件源的相对可靠性。给定退化观测 $y$、参考集合 $\mathcal{R}$ 与文本提示 $T$，一个实用的双先验恢复系统需要回答三个问题：当文本描述错误时，如何保留参考身份约束；当参考图像过时或误匹配时，如何避免错误纹理被采纳；当冲突只发生在眼镜、胡须或遮挡等局部属性上时，如何修正该局部而不牺牲整体身份。

本文提出 ConsistentFace，一个用于参考--文本双先验人脸恢复的冲突感知可信度分配适配器。它不重新设计扩散主干，而是在参考 token 和文本 token 进入去噪网络之前估计它们的可信度。具体而言，ConsistentFace 将可信度分配拆成三个互补决策：全局参考/文本偏好、属性槽级参考/文本偏好，以及外部先验整体不可靠时的保守回退。通过这种结构，模型可以在文本幻觉、参考失效、参考集污染、双先验联合失效和局部属性冲突之间采用不同路由，而不是用单一相似度或固定融合权重处理所有情况。

这一细粒度可靠性判断面临三个挑战。首先，参考--文本不一致只说明二者存在分歧，并不能说明哪一方错误。其次，严重退化图像本身也不总是可靠，模型不能简单地把退化观测当成裁判。第三，训练阶段可以知道冲突类型，但部署时不能要求用户提供“文本错误”或“参考错误”等标签。为此，ConsistentFace 使用多维证据区分错误来源，使用属性槽路由处理局部冲突，并使用相对排序监督训练权重方向；推理时，模型只根据 $(y,\mathcal{R},T)$ 自适应完成路由。

\textbf{本文的主要贡献如下：}

\begin{itemize}
    \item \textbf{提出身份保持双先验冲突建模：} 本文将参考图像和文本提示同时可用但可能冲突的人脸恢复场景表述为可信度分配问题，明确区分身份漂移、文本事实幻觉、参考失效、参考集合污染和局部属性冲突。
    
    \item \textbf{设计多证据与属性槽路由适配器：} 本文从参考--文本一致性、退化--先验一致性、参考集合内部一致性和退化强度中提取互补证据，并在头发、眼镜/眼睛、胡须/嘴巴、年龄/皮肤和配饰/遮挡五类属性槽上独立执行参考/文本分配和保守回退。

    \item \textbf{建立冲突监督和评测协议：} 本文使用相对权重排序、回退正则和解缠结正则训练路由行为，并通过同一主干比较、强双先验代理、匹配-$s_{RT}$ 反事实实验、跨评估器验证和人工评审检验增益是否来自可信度判断本身。

\end{itemize}


\section{相关工作}
\label{sec:main_related}
本节围绕与本文相关的三个研究维度展开论述：基于生成先验与个性化建模的人脸恢复、语言引导的恢复与编辑，以及多模态条件融合中的可靠性估计。尽管现有研究表明，生成先验、身份参考图像以及文本提示均能作为重要的额外线索，指导模型更准确地补全缺失的面部信息，但本文进一步关注如下问题：当多源先验同时存在且发生模态间冲突时，模型应如何动态判断其相对可信度，并执行细粒度的局部化权重分配。

\subsection{生成先验与个性化人脸恢复}
盲人脸恢复通常依赖强生成先验，以补全高度退化观测中缺失的高频细节。早期代表性工作如 GFPGAN \cite{wang2021gfpgan} 引入生成式人脸先验，以提升真实退化场景下的感知质量；RestoreFormer、VQFR 与 CodeFormer \cite{wang2022restoreformer,gu2022vqfr,zhou2022codeformer} 则通过键值记忆、向量量化字典或离散码本查询机制，在人脸结构严重受损时重建局部纹理。近年来，随着扩散模型（Diffusion Models）的发展，DR2、DifFace、DiffBIR、StableSR 等方法 \cite{wang2023dr2,yue2022difface,suin2024diffuse,gao2024diffmac,zhu2024visualstyle,min2024pdgrad,lu2024threepriors,chen2024bfrffusion,lin2024diffbir,wang2024stablesr,liang2024authface,zhu2022sgpn} 进一步将预训练扩散先验、区域自适应扩散、幻觉校正、视觉风格提示以及 3D 或梯度引导等机制引入恢复流程。此外，RestoreFormer++、DiffBFR、OSDFace 等工作 \cite{wang2023restoreformerpp,qiu2023diffbfr,wang2025osdface,do2025dynfacerestore,miao2025flipnet} 也从不同角度提升了复杂退化条件下的人脸恢复性能。

尽管上述方法在生成逼真细节方面取得了显著进展，但过强的生成先验也可能带来身份偏移（Identity Shift）风险。正如 PULSE \cite{menon2020pulse} 等研究所揭示的，恢复任务中存在固有的感知-失真权衡（Perception-Distortion Trade-off）：更高的视觉真实感往往可能以牺牲身份保真度为代价。为进一步约束恢复过程的解空间，参考引导恢复与个性化生成方法被相继提出。这类方法通过引入同一身份的参考图像、专用身份编码器、复合参考上下文或潜在扩散适配器 \cite{li2018gfrnet,li2020asffnet,li2022dmdnet,zhou2025referenceguided,varanka2024pfstorer,hsiao2024refldm,liu2025faceme} 来强化身份约束。例如，MyStyle \cite{nitzan2022mystyle} 表明，少量参考肖像即可为特定主体构建有效的生成约束；Textual Inversion、DreamBooth 等文本到图像个性化方法 \cite{gal2023textualinversion,ruiz2023dreambooth,kumari2023customdiffusion} 则将用户概念绑定至文本接口；InstantID 与 PuLID \cite{wang2024instantid,guo2024pulid} 进一步验证了面部身份嵌入与文本提示在扩散模型中的协同潜力。

上述方法说明，生成先验、身份参考和个性化适配器能够缩小人脸恢复的解空间，但也留下一个共同不足：外部先验通常被当作应尽量强化的条件，而较少被当作需要验证的证据。近期工作与本文最接近，但关注点仍有区别。CoFaDiff \cite{ming2026cofadiff} 主要讨论个性化生成中的身份保真度与文本一致性协调，核心对象是生成编辑而非退化观测约束下的恢复；IConFace \cite{niu2026iconface} 研究身份--结构不对称调节和无参考回退，但没有把参考、文本和退化观测之间的错误来源显式解耦；MCS \cite{li2025mcs} 与 A$^2$BFR \cite{zhu2026a2bfr} 强化了文本约束或属性感知恢复，却默认文本先验在目标属性上大体可信；FaceMe \cite{liu2025faceme} 强化参考身份约束，但不处理文本与参考同时可用且相互矛盾时的局部权重分配。本文因此把重点放在参考--文本双先验冲突下的可信度研判：模型不仅要利用先验，还要判断哪些先验值得采用。

\subsection{语言引导恢复与编辑}
自然语言为人脸恢复提供了具备可解释性与灵活性的控制接口，可描述年龄、发型、配饰、遮挡状态及局部面部特征等可变属性。相关研究探索了文本引导恢复、视觉风格提示学习以及通用指令驱动的图像恢复 \cite{xie2023tfrgan,an2026tbfr,zhu2024visualstyle,tao2025mgfr,conde2024instructir,qi2024spire}。这些研究表明，语言条件的引入能够提升恢复结果的可控性，并允许用户以较低交互成本指定局部属性的重建目标。

然而，语言先验固有的开放性也引入了不确定性。文本提示可能源自自动图像描述、过时人工描述或视觉语言模型生成的描述；在图像恢复和指令编辑任务中，提示若包含错误属性，模型可能仍会生成视觉合理但身份或属性偏移的结果 \cite{xie2023tfrgan,conde2024instructir,qi2024spire,tao2025mgfr}。因此，ConsistentFace 的设计目标并非削弱文本引导，而是在多模态先验并存的复杂场景下显式评估语言条件的可靠性。该设计支持模型在宏观身份层面与微观属性槽层面分别进行权重分配，即动态采纳、校准或抑制文本信息，而非全局性提高或降低文本条件权重。

\subsection{多模态可靠性与可信度分配}
在多模态条件融合领域，融合门控（Gating）、不确定性加权以及基于 CLIP 的一致性评分是处理模态协同的常用手段，相关思想也广泛出现在图文生成、图像编辑和条件扩散模型中 \cite{radford2021clip,zhang2023controlnet,ye2023ipadapter,goel2024pairdiffusion}。这类方法提供了融合多个条件的通用工具，但通常不区分“文本错误”“参考错误”和“两个先验同时不可靠”等错误来源。IP-Adapter \cite{ye2023ipadapter}、PhotoMaker \cite{li2024photomaker}、InstantID \cite{wang2024instantid} 和 PuLID \cite{guo2024pulid} 等身份适配器进一步增强了图像条件注入能力，但它们主要回答“如何更强地注入身份”，而不是“当身份参考与文本提示冲突时应如何判断谁更可信”。

基于上述观察，ConsistentFace 的核心主张并非构造更复杂的通用门控网络，也不是简单叠加强身份适配器，而是为分配变量赋予明确的可信度语义和监督约束：多维证据解耦用于定位冲突来源，保守回退为双先验同时失效提供低风险路径，细粒度属性槽则将局部属性决策从全局身份决策中解耦出来。为验证这一点，本文将 ConsistentFace 与通用额外 token 条件、属性槽打乱、纯参考--文本一致性分配器、无排序监督分配器以及无回退分配器进行消融对比。由于参考 token 的构建使用 ArcFace 特征 \cite{deng2019arcface}，若只用 ArcFace 评估身份，模型输入特征与评价指标会发生耦合；因此本文同时报告 AdaFace、MagFace 与 CurricularFace \cite{kim2022adaface,meng2021magface,huang2020curricularface}，并结合客观属性评估器与人工盲测，将身份与属性可靠性纳入评价体系。

\section{问题表述}

给定清晰目标人脸 $x$，令 $y=\mathcal{D}(x)$ 表示经过未知退化算子 $\mathcal{D}$ 得到的观测图像，其中退化可包含模糊、下采样、噪声、压缩和光照变化。双先验人脸恢复同时接收退化观测 $y$、参考图像集合 $\mathcal{R}$ 和自然语言提示 $T$，并输出恢复结果 $\hat{x}=G_{\theta}(y,\mathcal{R},T)$。与单一参考引导恢复不同，该设定中的不确定性来自三处：观测图像可能过弱，参考图像可能失效，文本提示也可能携带错误属性。

本文将自然语言提示解耦为结构化属性槽 $T=\{T_a:a\in\mathcal{A}\}$ 和自由文本。其中，$\mathcal{A}$ 包含头发、眼镜/眼睛、胡须/嘴巴、年龄/皮肤和配饰/遮挡五类常见面部属性。选择这五类属性并不是为了覆盖所有开放世界语义，而是因为它们在恢复提示中出现频率高、容易通过自动评估或人工盲测检查，并且常与身份特征发生耦合。

双先验冲突对应三个不同决策。第一，模型需要在参考身份 token 和文本语义 token 之间分配全局偏好，以处理“参考可靠但文本幻觉”或“文本可靠但参考过时”等宏观冲突。第二，当参考与文本同时可疑时，模型需要降低外部条件强度，更多依赖退化观测和恢复先验。第三，模型需要在局部属性槽上独立决策，避免一个错误眼镜或发型描述触发全局身份条件丢弃。为此，后文用 $\alpha$ 表示全局参考/文本偏好，用 $c$ 表示外部先验整体置信度，并用 $\alpha_a,c_a$ 表示属性槽 $a$ 的局部偏好和局部回退置信度。

训练阶段，每个样本带有冲突类型标签 $r$。该标签只用于构造排序监督和槽级污染集合，不作为模型输入，也不在推理阶段出现。推理阶段，模型只能根据 $(y,\mathcal{R},T)$ 计算证据并预测权重。这样的设定避免了冲突标签泄漏，也使评测更接近真实使用场景。

基于上述任务定义，冲突感知恢复系统应具备四类行为：1）当文本准确而参考过时或失配时，利用文本校正可变属性；2）当参考可靠而文本存在事实幻觉时，抑制错误语义；3）当参考与文本联合失效时，衰减外部先验并启用保守回退；4）当冲突仅局限于单一局部属性时，只调整对应属性槽，保留其他仍可信的文本信息。

这些行为也决定了实验验证准则。第一，在事实幻觉或强冲突提示下，方法应提高身份保真度，而不是简单丢弃所有文本条件。第二，方法应提升局部目标属性，同时降低错误先验采纳率。第三，收益应在同一主干、强双先验代理和跨评估器设置下保持稳定，以排除模型容量、调参幅度或单一指标偏好造成的混淆。

具体实验中，本文使用预定义冲突清单实例化问题空间，覆盖匹配提示、轻/重度文本冲突、事实幻觉、过时参考、错误参考和双先验失效。成对样本控制退化难度、参考质量、提示长度和检查点选择。除模板化冲突外，实验还引入自然语言和长自由文本提示，用于检验模型是否能在更接近日常描述的输入中保持可靠性判断。

\section{方法}
\label{sec:main_method}

ConsistentFace 建立在 SDXL/ControlNet 恢复主干之上。退化图像通过 ControlNet 提供结构约束；参考图像通过身份编码器和图像提示适配器思想转化为身份 token；文本提示通过 SDXL 文本编码器转化为语义 token。本文的新增模块是一个前置可信度适配器，它在条件进入扩散去噪网络之前完成三步处理。第一，提取结构、身份和文本特征。第二，计算跨模态一致性、观测--先验一致性、参考集合内部稳定性和退化强度等证据。第三，生成全局条件、属性槽条件和保守回退 token，并将它们送入 SDXL/ControlNet 去噪路径。图~\ref{fig:multi_evidence_routing} 给出可信度分配适配器，图~\ref{fig:overview} 给出完整恢复流程。

\subsection{条件特征抽取}
对于参考集合 $\mathcal{R}=\{R_i\}_{i=1}^{M}$，模型使用身份编码器和视觉编码器提取参考特征，并将其映射为可注入扩散模型的身份 token：
\begin{equation}
\label{eq:reference_tokens}
\begin{aligned}
e^{I}_{R_i}&=E_I(R_i),\\
Z_{R_i}&=E_V(R_i),\\
V&=\Phi_R\!\left(\{e^{I}_{R_i},Z_{R_i}\}_{i=1}^{M}\right),\\
V&=[v_1,\ldots,v_K].
\end{aligned}
\end{equation}
式~\eqref{eq:reference_tokens} 中，$R_i$ 表示第 $i$ 张参考图像，$i$ 是参考图像索引，$M$ 是参考图像数量；$E_I$ 表示身份编码器（实现中采用 ArcFace），$e^{I}_{R_i}$ 是参考图像 $R_i$ 的身份嵌入；$E_V$ 表示视觉特征编码器，$Z_{R_i}$ 是参考图像的视觉特征；$\Phi_R$ 是可学习的参考特征映射器，用于将多张参考图像的身份和视觉特征聚合为 token 序列；$V$ 表示参考身份 token 块；$v_k$ 表示第 $k$ 个身份 token，$k$ 是 token 索引，$K$ 是身份 token 数量。

文本提示 $T$ 通过文本编码器得到序列语义特征，并映射到与身份 token 兼容的条件空间：
\begin{equation}
\label{eq:text_tokens}
\begin{aligned}
H&=E_T(T),\\
t&=\operatorname{Pool}(H),\\
U&=\Phi_T(H)=[u_1,\ldots,u_K].
\end{aligned}
\end{equation}
式~\eqref{eq:text_tokens} 中，$T$ 表示自然语言提示；$E_T$ 表示 SDXL 文本编码器；$H$ 表示文本 token 序列特征；$\operatorname{Pool}(\cdot)$ 表示池化操作；$t$ 表示全局文本语义向量；$\Phi_T$ 表示可学习的文本特征映射器；$U$ 表示文本语义 token 块；$u_k$ 表示第 $k$ 个文本语义 token。为便于后续计算，$U$ 与 $V$ 被映射到相同维度并具有相同 token 数量 $K$。

本文还使用退化图像编码器提取观测特征，并定义参考和文本 token 的平均表示：
\begin{equation}
\label{eq:mean_tokens}
f_y=E_y(y),\qquad
\bar{v}=\frac{1}{K}\sum_{k=1}^{K}v_k,\qquad
\bar{u}=\frac{1}{K}\sum_{k=1}^{K}u_k.
\end{equation}
式~\eqref{eq:mean_tokens} 中，$y$ 表示退化观测图像；$E_y$ 表示退化图像编码器；$f_y$ 表示由 $y$ 提取的观测特征；$\bar{v}$ 表示身份 token 的平均特征；$\bar{u}$ 表示文本语义 token 的平均特征；$1/K$ 是归一化系数，用于对 $K$ 个 token 求均值。

\subsection{多维可信度证据}
为避免将所有冲突简化为单一参考--文本相似度，本文首先定义余弦相似度：
\begin{equation}
\label{eq:cosine_similarity}
\operatorname{sim}(a,b)=
\frac{a^{\top}b}{\|a\|_2\,\|b\|_2}.
\end{equation}
式~\eqref{eq:cosine_similarity} 中，$a$ 和 $b$ 表示任意两个待比较的特征向量；$a^{\top}b$ 表示二者的内积；$\|\cdot\|_2$ 表示二范数；$\operatorname{sim}(a,b)$ 的取值越大，表示两个特征在方向上越一致。

基于该相似度，ConsistentFace 构造四类全局证据：
\begin{subequations}
\label{eq:multi_evidence_scores}
\begin{align}
s_{RT}
&=\operatorname{sim}\!\left(P_R(\bar{v}),P_T(t)\right),\\
s_{yR}
&=\operatorname{sim}\!\left(P_y^{I}(f_y),P_R(\bar{v})\right),\\
s_{yT}
&=\operatorname{sim}\!\left(P_y^{A}(f_y),P_T^{A}(t)\right),\\
s_{RR}
&=\frac{2}{M(M-1)}
\sum_{1\leq i<j\leq M}
\operatorname{sim}\!\left(e^{I}_{R_i},e^{I}_{R_j}\right).
\end{align}
\end{subequations}
式~\eqref{eq:multi_evidence_scores} 中，$s_{RT}$ 表示参考先验与文本先验的一致性；$s_{yR}$ 表示退化观测与参考集合的身份一致性；$s_{yT}$ 表示退化观测与文本属性的一致性；$s_{RR}$ 表示参考集合内部的身份稳定性。$P_R$、$P_T$、$P_y^{I}$、$P_y^{A}$ 和 $P_T^{A}$ 均为可学习投影头，分别将参考特征、文本特征、退化图像身份特征、退化图像属性特征和文本属性特征映射到可比较空间。$e^{I}_{R_i}$ 与 $e^{I}_{R_j}$ 分别表示第 $i$ 和第 $j$ 张参考图像的身份嵌入；求和条件 $1\leq i<j\leq M$ 表示遍历所有不同参考图像对；系数 $2/[M(M-1)]$ 用于对所有参考图像对的相似度取平均。若只有一张参考图像，$s_{RR}$ 在实现中被设为中性常数，避免参考集合内部一致性项影响判断。

除上述一致性证据外，模型还估计退化强度：
\begin{equation}
\label{eq:degradation_score}
q=Q(y).
\end{equation}
式~\eqref{eq:degradation_score} 中，$Q$ 表示退化强度估计器，$q$ 表示由退化图像 $y$ 得到的退化强度代理分数。$q$ 越大，表示观测图像可靠性越低，外部先验权重分配应更多考虑保守回退路径。

最终的全局证据向量定义为
\begin{equation}
\label{eq:global_evidence_vector}
g=
\left[
s_{RT},\,
s_{yR},\,
s_{yT},\,
s_{RR},\,
q,\,
s_{RT}-s_{yR},\,
s_{RT}-s_{yT}
\right].
\end{equation}
式~\eqref{eq:global_evidence_vector} 中，$g$ 表示全局可信度证据向量；前五项分别保留参考--文本一致性、退化--参考一致性、退化--文本一致性、参考集合内部一致性和退化强度；差值 $s_{RT}-s_{yR}$ 用于区分“参考与文本一致但二者都偏离退化身份”和“参考与退化身份一致但文本不可靠”的情形；差值 $s_{RT}-s_{yT}$ 用于辅助判断文本属性是否与退化观测一致。

\subsection{全局可信度研判与权重分配}
全局可信度分配器根据证据向量 $g$ 预测两个标量权重：
\begin{subequations}
\label{eq:global_weights}
\begin{align}
\alpha&=\sigma\!\left(f_{\alpha}(g)\right),\\
c&=\sigma\!\left(f_{c}(g)\right).
\end{align}
\end{subequations}
式~\eqref{eq:global_weights} 中，$f_{\alpha}$ 和 $f_c$ 表示两个独立的多层感知机；$\sigma(\cdot)$ 表示 Sigmoid 函数，用于将输出压缩到 $[0,1]$；$\alpha$ 表示视觉身份先验与文本语义先验之间的分配权重，$\alpha$ 越大表示参考身份 token 权重越高，$1-\alpha$ 越大表示文本语义 token 权重越高；$c$ 表示外部先验整体置信度，$c$ 越大表示参考和文本条件权重越高，$1-c$ 越大表示保守回退 token 权重越高。

全局条件 token 定义为
\begin{equation}
\label{eq:global_fusion}
F_g=
c\Big(\alpha V+(1-\alpha)U\Big)
+(1-c)B_g.
\end{equation}
式~\eqref{eq:global_fusion} 中，$F_g$ 表示全局融合 token 块；$V$ 是参考身份 token 块；$U$ 是文本语义 token 块；$B_g$ 是可学习的全局保守回退 token，与 $V$ 和 $U$ 具有相同维度；$\alpha$ 和 $c$ 为式~\eqref{eq:global_weights} 中得到的标量权重，并在 token 维度上广播。当参考可靠而文本可疑时，期望 $\alpha$ 增大；当文本可靠而参考过时或失配时，期望 $\alpha$ 减小；当参考和文本均不可靠时，期望 $c$ 减小，从而降低外部先验的条件强度。

\subsection{属性槽级可信度分配}
全局权重只能给出整体偏好，难以处理“文本提示仅在眼镜属性上存在错误，而发型描述仍然正确”这类局部冲突。因此，本文将提示拆分为属性槽集合 $\mathcal{A}$，其中 $\mathcal{A}$ 包含头发、眼镜/眼睛、胡须/嘴巴、年龄/皮肤和配饰/遮挡五类属性。对于任一属性槽 $a\in\mathcal{A}$，模型从文本序列中抽取槽级文本特征：
\begin{equation}
\label{eq:slot_text_feature}
t_a=\Psi_a(H).
\end{equation}
式~\eqref{eq:slot_text_feature} 中，$a$ 表示属性槽索引；$\Psi_a$ 表示属性槽 $a$ 对应的文本特征提取器；$H$ 是式~\eqref{eq:text_tokens} 中的文本序列特征；$t_a$ 表示属性槽 $a$ 的文本语义 token。

为了获得与属性槽相关的参考特征，模型使用可学习查询对身份 token 执行注意力汇聚：
\begin{subequations}
\label{eq:slot_attention}
\begin{align}
\omega_{a,k}
&=
\frac{
\exp\!\left((W_Q e_a)^{\top}(W_K v_k)/\sqrt{d}\right)
}{
\sum_{\ell=1}^{K}
\exp\!\left((W_Q e_a)^{\top}(W_K v_{\ell})/\sqrt{d}\right)
},\\
v_a
&=
\sum_{k=1}^{K}\omega_{a,k}W_V v_k.
\end{align}
\end{subequations}
式~\eqref{eq:slot_attention} 中，$e_a$ 表示属性槽 $a$ 的可学习查询向量；$W_Q$、$W_K$ 和 $W_V$ 分别表示查询、键和值的线性投影矩阵；$d$ 表示注意力特征维度，$\sqrt{d}$ 用于缩放点积；$\omega_{a,k}$ 表示属性槽 $a$ 对第 $k$ 个身份 token 的注意力权重；$\ell$ 是归一化求和中的 token 索引；$v_a$ 表示属性槽 $a$ 对应的参考视觉特征。

属性槽级证据由参考--文本槽一致性和退化--文本槽一致性构成：
\begin{subequations}
\label{eq:slot_scores}
\begin{align}
s_a
&=\operatorname{sim}\!\left(P_R^{a}(v_a),P_T^{a}(t_a)\right),\\
s_{ya}
&=\operatorname{sim}\!\left(P_y^{a}(f_y),P_T^{a}(t_a)\right).
\end{align}
\end{subequations}
式~\eqref{eq:slot_scores} 中，$s_a$ 表示属性槽 $a$ 上参考特征与文本特征的一致性；$s_{ya}$ 表示属性槽 $a$ 上退化观测与文本特征的一致性；$P_R^{a}$、$P_T^{a}$ 和 $P_y^{a}$ 是属性槽 $a$ 对应的可学习投影头；$v_a$ 是槽级参考特征，$t_a$ 是槽级文本特征，$f_y$ 是退化图像特征。

随后，模型为每个属性槽预测独立的视觉/文本分配权重和回退置信度：
\begin{subequations}
\label{eq:slot_weights}
\begin{align}
g_a&=\left[g,\,s_a,\,s_{ya},\,\alpha,\,c\right],\\
\alpha_a&=\sigma\!\left(f_{\alpha}^{a}(g_a)\right),\\
c_a&=\sigma\!\left(f_{c}^{a}(g_a)\right).
\end{align}
\end{subequations}
式~\eqref{eq:slot_weights} 中，$g_a$ 表示属性槽 $a$ 的局部证据向量，由全局证据 $g$、槽一致性分数 $s_a$、退化--文本槽一致性 $s_{ya}$、全局视觉/文本权重 $\alpha$ 和全局外部先验置信度 $c$ 拼接而成；$f_{\alpha}^{a}$ 和 $f_c^{a}$ 表示属性槽 $a$ 的两个多层感知机；$\alpha_a$ 表示属性槽 $a$ 中参考视觉特征与文本特征的分配权重；$c_a$ 表示属性槽 $a$ 的外部先验置信度。

属性槽级融合 token 定义为
\begin{equation}
\label{eq:slot_fusion}
F_a=
c_a\Big(\alpha_a v_a+(1-\alpha_a)t_a\Big)
+(1-c_a)B_a.
\end{equation}
式~\eqref{eq:slot_fusion} 中，$F_a$ 表示属性槽 $a$ 的融合 token；$v_a$ 表示槽级参考视觉特征；$t_a$ 表示槽级文本特征；$B_a$ 表示属性槽 $a$ 的保守回退 token；$\alpha_a$ 控制该属性槽中参考与文本的相对权重；$c_a$ 控制该属性槽中外部先验与保守回退之间的相对权重。所有条件 token 最终拼接为
\begin{equation}
\label{eq:condition_tokens}
F=\operatorname{Concat}\!\left(F_g,\{F_a:a\in\mathcal{A}\}\right).
\end{equation}
式~\eqref{eq:condition_tokens} 中，$F$ 表示最终注入扩散模型的条件 token 集合；$\operatorname{Concat}(\cdot)$ 表示 token 维度上的拼接操作；$\{F_a:a\in\mathcal{A}\}$ 表示遍历属性集合 $\mathcal{A}$ 后得到的所有槽级 token。

\subsection{训练目标与推理}
扩散训练采用标准噪声预测目标。设 $z_0$ 为清晰目标图像 $x$ 的 VAE 潜变量，$\tau$ 为扩散时间步，$\epsilon$ 为从标准高斯分布采样的噪声，则加噪潜变量为
\begin{equation}
\label{eq:noisy_latent}
z_{\tau}
=
\sqrt{\bar{\gamma}_{\tau}}\,z_0
+
\sqrt{1-\bar{\gamma}_{\tau}}\,\epsilon.
\end{equation}
式~\eqref{eq:noisy_latent} 中，$z_{\tau}$ 表示第 $\tau$ 步的带噪潜变量；$\bar{\gamma}_{\tau}$ 表示扩散噪声调度在时间步 $\tau$ 的累计保留系数；$z_0$ 表示干净图像的潜空间表示；$\epsilon$ 表示随机噪声。

对应的扩散去噪损失为
\begin{equation}
\label{eq:diffusion_loss}
\mathcal{L}_{\mathrm{diff}}
=
\mathbb{E}_{z_0,\epsilon,\tau}
\left[
\left\|
\epsilon-\epsilon_{\theta}(z_{\tau},\tau,C_y,F)
\right\|_2^2
\right].
\end{equation}
式~\eqref{eq:diffusion_loss} 中，$\mathcal{L}_{\mathrm{diff}}$ 表示扩散噪声预测损失；$\mathbb{E}$ 表示对训练样本、噪声和时间步的期望；$\epsilon_{\theta}$ 表示参数为 $\theta$ 的去噪网络；$C_y$ 表示由退化图像 $y$ 提取的 ControlNet 结构条件；$F$ 是式~\eqref{eq:condition_tokens} 中的可信度分配后条件 token；$\|\cdot\|_2^2$ 表示平方二范数。

排序监督来自训练清单中的冲突类型标签 $r$，但该标签只用于构造权重之间的相对偏序，不作为推理输入，也不直接拼接到证据向量。令全局权重集合为
\begin{equation}
\label{eq:global_weight_set}
\mathcal{W}_g=\{\alpha,\ 1-\alpha,\ c,\ 1-c\},
\end{equation}
其中四项分别对应参考偏好、文本偏好、外部先验置信度和保守回退偏好。每个冲突类型 $r$ 预先映射到一组期望排序对 $\mathcal{P}_g(r)=\{(w^+,w^-)\}$。例如，文本事实幻觉要求 $\alpha>1-\alpha$ 且 $c>1-c$；错误或过时参考要求 $1-\alpha>\alpha$ 且 $c>1-c$；双先验联合失效要求 $1-c>c$。全局排序损失定义为
\begin{equation}
\label{eq:global_rank_loss}
\begin{aligned}
\mathcal{L}^{g}_{\mathrm{rank}}
=&
\frac{1}{|\mathcal{P}_g(r)|}
\sum_{(w^+,w^-)\in\mathcal{P}_g(r)}
\\[-1pt]
&\max\!\left(0,\ m_g-(w^+-w^-)\right).
\end{aligned}
\end{equation}
其中 $m_g$ 是全局排序间隔。该损失只约束相对方向，不要求权重拟合人工设定的固定常数。

对于属性槽监督，训练清单同时记录被污染的局部属性集合 $\mathcal{A}^{-}(r)\subseteq\mathcal{A}$ 以及该槽应优先参考、优先文本或回退的目标路由。令槽级权重集合为
\begin{equation}
\label{eq:slot_weight_set}
\mathcal{W}_a=\{\alpha_a,\ 1-\alpha_a,\ c_a,\ 1-c_a\}.
\end{equation}
槽级排序对 $\mathcal{P}_a(r,a)$ 仅在 $a\in\mathcal{A}^{-}(r)$ 时启用；无冲突槽不施加强排序，以避免破坏仍可信的文本指导。槽级排序损失为
\begin{equation}
\label{eq:slot_rank_loss}
\begin{aligned}
\mathcal{L}^{a}_{\mathrm{rank}}
=&
\frac{1}{|\mathcal{A}^{-}|}
\sum_{a\in\mathcal{A}^{-}}
\frac{1}{|\mathcal{P}_a(r,a)|}
\\[-1pt]
&\sum_{(w_a^+,w_a^-)\in\mathcal{P}_a(r,a)}
\max\!\left(0,\ m_a-(w_a^+-w_a^-)\right).
\end{aligned}
\end{equation}
其中 $m_a$ 是槽级排序间隔。当 $\mathcal{A}^{-}$ 为空时，该项置为 $0$。

保守回退正则用于双先验均可疑时提高回退 token 的使用比例。设 $b_g(r)\in\{0,1\}$ 表示全局是否需要回退，$b_a(r,a)\in\{0,1\}$ 表示属性槽是否需要回退，则
\begin{equation}
\label{eq:fallback_loss}
\begin{aligned}
\mathcal{L}_{\mathrm{fb}}
=&
-b_g\log(1-c)-(1-b_g)\log c\\
&+
\frac{1}{|\mathcal{A}|}
\sum_{a\in\mathcal{A}}
\left[-b_a\log(1-c_a)-(1-b_a)\log c_a\right].
\end{aligned}
\end{equation}
实现中对 $\log(\cdot)$ 的输入加入 $10^{-6}$ 下界以避免数值不稳定。

为降低模型记忆提示模板、捐赠者身份或退化族而非真实证据关系的风险，本文使用解缠结正则约束证据向量与非目标元信息 $\xi$ 的批内相关性。令 $\tilde{g}$ 和 $\tilde{g}_a$ 表示批内标准化后的全局与槽级证据，$\xi$ 为提示模板、退化族和参考来源标识的一热向量，则
\begin{equation}
\label{eq:regularization_loss}
\mathcal{L}_{\mathrm{reg}}
=
\left\|\operatorname{Cov}(\tilde{g},\xi)\right\|_F^2
+
\frac{1}{|\mathcal{A}|}
\sum_{a\in\mathcal{A}}
\left\|\operatorname{Cov}(\tilde{g}_a,\xi)\right\|_F^2.
\end{equation}
该正则不惩罚与冲突类型 $r$ 的必要排序关系，而是抑制模型利用模板或数据来源捷径完成路由。

完整训练目标为
\begin{equation}
\label{eq:training_objective}
\mathcal{L}
=
\mathcal{L}_{\mathrm{diff}}
+
\lambda_g\mathcal{L}^{g}_{\mathrm{rank}}
+
\lambda_a\mathcal{L}^{a}_{\mathrm{rank}}
+
\lambda_b\mathcal{L}_{\mathrm{fb}}
+
\lambda_r\mathcal{L}_{\mathrm{reg}}.
\end{equation}
式~\eqref{eq:training_objective} 中，$\mathcal{L}$ 表示总损失；$\lambda_g$、$\lambda_a$、$\lambda_b$ 和 $\lambda_r$ 是四个非负超参数，分别控制全局排序、槽级排序、回退正则和解缠结正则。主实验使用同一组超参数训练所有 ConsistentFace 变体，并在验证集上固定后再评估自然语言、自由文本、评估器替换和人工评审子集。

推理阶段，模型只接收 $(y,\mathcal{R},T)$，不再使用冲突类型标签、属性污染标签或模板编号。全局证据 $g$ 和槽级证据 $g_a$ 由输入本身计算得到，随后生成 $F_g$、$F_a$ 和最终条件 token $F$。因此，ConsistentFace 在部署时不依赖人工指定参考或文本的优先级，而是根据退化观测、参考集合和文本提示之间的证据关系自适应完成可信度研判与权重分配。

\section{实验}
\label{sec:main_experiments}

\subsection{评测设置}
实验回答两个问题。第一，在不同输入接口下，双先验冲突恢复为什么需要同时报告图像质量、身份保真度、属性跟随和事实幻觉抑制能力。第二，在主干、输入和训练清单固定时，ConsistentFace 的收益是否来自可信度判断，而不是来自额外参数、调参幅度或某个评估器偏好。前者由跨范式基准说明任务背景，后者由同一主干比较、强双先验代理和消融实验承担核心检验。

所有同一主干实验使用相同的 $(y,\mathcal{R},T)$、SDXL/ControlNet 恢复主干、参考编码器、训练清单和推理步数。比较对象包括普通 token 融合、不确定性加权、参考--文本双分支、CLIP/ArcFace 一致性分配器、多证据无排序监督分配器、无回退分配器，以及仅使用 $\alpha=f(s_{RT})$ 的 RT-only ConsistentFace。身份指标取 ArcFace、AdaFace、MagFace 和 CurricularFace 类识别器的平均值，以减少对单一 ArcFace 编码器的依赖。TAR@FAR 和排名反转用于检查传统恢复指标是否掩盖身份或属性错误。属性指标包括 CelebA 训练分类器、OpenCLIP 属性评分、开放属性解析器和人工评审子集。除特别说明外，所有表格均报告预定义冲突清单上的实测结果。

\subsubsection{数据构建和划分}
评估套件固定目标身份、退化观测、同一身份参考图像和冲突提示。退化类型采样自近期人脸恢复工作常用退化族，冲突来源通过身份实例和提示实例控制。五个随机种子的主要比较使用完整冲突多样化套件。自然语言和自由文本子集用更长、更接近日常描述的提示替换模板提示，并额外纳入疤痕、面具或手部遮挡、少见配饰、跨年龄参考以及多个局部属性同时矛盾等情形。

为区分“参考--文本不一致”背后的错误来源，本文构造匹配 $s_{RT}$ 和 $q$ 的反事实划分。该划分成对生成事实幻觉和错误参考样本，使二者具有近似相同的参考--文本一致性和退化强度分布，同时平衡退化族、参考数量、参考质量、身份划分和提示长度。该设置用于检验 $s_{yR}$、$s_{yT}$ 和 $s_{RR}$ 是否提供了足以改变 $\alpha$ 分配方向的额外信息。所有冲突标签均在训练前固定；事实幻觉提示从目标和参考集合之外的捐赠者身份采样，并与方法设计、阈值选择和检查点选择隔离。

\subsubsection{决策指标}
主要身份差距定义为匹配条件与事实幻觉条件之间的身份相似度下降，并在四个
识别器上取平均，以降低参考 token 中 ArcFace 特征带来的评估耦合。
TAR@FAR 衡量恢复图像是否跨过预设误接受率下的验证阈值；排名反转衡量
PSNR、SSIM 或 LPIPS 等传统恢复指标是否会偏好身份或属性质量更差的候选结果。
\begin{center}
\setlength{\fboxsep}{4pt}
\fbox{\begin{minipage}{0.96\linewidth}
\small
\setlength{\abovedisplayskip}{3pt plus 1pt minus 1pt}
\setlength{\belowdisplayskip}{3pt plus 1pt minus 1pt}
\textbf{指标定义。}
给定身份相似度 $S_h$，身份差距定义为
\[
\begin{aligned}
\mathrm{IdentityGap}
=\mathbb{E}_{h}\big[
&\mathbb{E}_{m}S_h(\hat{x}^{m},x)\\
&-\mathbb{E}_{c}S_h(\hat{x}^{c},x)
\big],
\end{aligned}
\]
其中 $m$ 和 $c$ 分别表示匹配条件与冲突条件。
对于候选集 $\mathcal{C}_i$，排序反转定义为
\[
\begin{aligned}
\mathrm{Rev}_i
&=\mathbf{1}\{Q(k^\star)<Q(j)-\tau,\ \exists j\},\\
k^\star
&=\arg\max_{k\in\mathcal{C}_i}M(k).
\end{aligned}
\]
其中 $M\in\{\mathrm{PSNR},\mathrm{SSIM},-\mathrm{LPIPS}\}$，
$Q$ 是身份或目标属性质量，$\tau=10^{-4}$ 是验证平局边距，
精确平局将被忽略。槽位 $a$ 上的错误先验采用定义为
\[
\begin{aligned}
\mathrm{WPA}_a
=\mathbf{1}\{&
E_a(\hat{x})=z_a^{\mathrm{wrong}}\\
&\land E_a(\hat{x})\ne z_a^{\mathrm{target}}\}.
\end{aligned}
\]
该指标使用预设评估器阈值验证。局部反转将相同选择器应用于局部冲突候选，
其中 $Q$ 由局部 Target Attr-F1 和错误先验拒绝共同定义。
\end{minipage}}
\end{center}
这些实用边际来自验证集上的决策敏感性分析，并非通用的人脸恢复阈值：
$15\%$ 的相对身份差距降低、$0.015$ 的 Major Attr-F1、$0.005$ 的
TAR 下降减小，以及五个百分点的排序反转变化，是验证集中至少百分之五
身份样本或属性检查槽位会改变提示策略或验证决策的最小变化量。

\subsection{同一主干比较}
\subsubsection{基线构造}
同一主干基线用于排除更简单的替代解释。token 级融合检验“暴露更多 token”
是否已经足够；参考--文本分支保留个性化参考通路，同时加入可训练的语言引导
分支；CLIP/ArcFace 一致性分配器检验语义--图像一致性和身份识别一致性是否可以
替代可信度监督；多证据无排序监督分配器使用与 ConsistentFace 相同的
$g_\alpha$ 特征，但移除权重排序损失，从而区分输入证据与监督信号的作用；
无回退分配器接受可信度监督，但在两个外部先验都可疑时不能选择保守 token $B$。

所有基线使用相同退化图像、参考集合、文本提示、优化器计划、扩散步数和
检查点选择规则。可训练适配器容量与 ConsistentFace 的预算宽度匹配，
扩散主干、VAE、文本编码器、参考编码器和采样器保持固定。检查点在查看
保留冲突类型、多样化扰动、自然语言/自由文本提示以及评估器替换和人工评审结果之前
确定。换言之，主要表格比较的是同一恢复系统内部的可信度分配机制，而不是直接比较多个
调参程度不同的现成管线。若通用融合、一致性分配器、文本分支或无回退分配器
在该设置下仍然落后，性能增益就更可能来自结构化可信度分配设计。

\subsubsection{指标解耦和人工评审}
由于 ArcFace 特征参与参考 token 构造，单独使用 ArcFace 分数作为主身份
指标并不稳妥。因此，主要表格报告 ArcFace、AdaFace、MagFace 和 CurricularFace
类识别器的平均身份差距，仅 ArcFace 分析放在补充材料中。不同识别器在
化妆、弱光和跨年龄参考下可能给出不同排序；平均差距要求多个识别器家族
方向一致，而不是只匹配某一个编码器。TAR@FAR 和 Rank-1 检索进一步把
嵌入相似度转化为验证式决策和检索排序。属性评估同样采用冗余设计：
CelebA 分类器提供稳定脚本化指标，OpenCLIP 属性评分检验语言--图像相似性，
开放属性解析器提供第三个自动视角，人工评审则覆盖自动评估器较脆弱的
局部冲突，如年龄/皮肤、配饰或妆容、发型、眼镜、面具、手部和遮挡。我们
关注不同评估器下的相对排序，而不只报告单个绝对 Attr-F1。

\subsection{可信度分配与错误来源区分}
本节检验模型能否区分“文本错误”和“参考错误”。二者都会表现为参考--文本不一致，因此 RT-only 变体只能感知冲突程度，不能判断错误来源。引入 $s_{yR}$、$s_{yT}$、$s_{RR}$ 和 $q$ 后，多维证据分配器可以在匹配的 $s_{RT},q$ 下让视觉权重和文本权重向相反方向变化。分布匹配的反事实结果验证了这一点：当 $s_{RT}$ 和 $q$ 被匹配后，RT-only 变体接近随机水平（TH/VD AUROC $0.55/0.53$），而完整证据向量达到 $0.82/0.84$，并将自然语言/自由文本子集中的错误先验采用率降至 $0.158$。这些结果并不把 $s_{yR}$ 或 $s_{yT}$ 当作理想裁判，而是说明在平衡退化族与参考质量后，退化图像线索提供了 RT-only 变体没有观察到的信息。

\section{局限性和结论}
\label{sec:main_conclusion}

ConsistentFace 的目标是提高恢复过程中的先验使用可靠性，而不是提供身份认证。它依赖三个前提：训练清单覆盖常见冲突类型，五类属性槽能够描述主要局部矛盾，退化观测和参考集合仍保留可用于判断的证据。当输入包含开放属性、多身份参考、严重遮挡、极端低分辨率、跨年龄跨度过大或长文本混合矛盾时，这些前提会被削弱。

主结果中完整模型仍有 44.6\% 的排名反转率。这一负面结果说明，即使可信度分配显著降低了错误先验采纳，传统画质指标仍可能偏好身份或属性更差的候选结果。图~\ref{fig:failure_modes_limitations} 将剩余错误归纳为严重退化、遮挡、过时参考、弱属性槽和长文本组合矛盾。后续工作需要更强的不确定性校准、遮挡感知身份置信度、时间感知参考加权以及更细粒度的人审属性标注。

在上述范围内，本文结论是：多维证据可以为先验错误来源提供可区分线索，保守回退可以避免两个先验均可疑时强制采纳外部信息，属性槽级分配可以定位常见局部冲突。该组合在同一主干分配基线之上改善了身份--属性权衡；但输出仍应被视为恢复候选图像，不能被解释为目标身份的真实性证明。

\subsection{伦理与发布说明}
身份保持恢复可用于历史照片修复、辅助媒体增强和稳健性基准测试，但恢复结果不应在门禁、执法、金融开户、招聘筛查或身份核验等高风险场景中作为身份证据。为了降低误用风险，模型发布应同时提供输入退化记录、参考来源摘要、可信度诊断和失败模式说明，而不是只发布视觉上最清晰的恢复图。数据发布需遵守原始数据许可、肖像权和隐私约束；当原始人脸图像无法再分发时，只发布提示、评测清单、哈希值、聚合统计和可复现实验脚本。对于涉及未成年人、敏感身份或不可撤回同意的数据，应排除原图分发并限制模型权重的使用场景。
